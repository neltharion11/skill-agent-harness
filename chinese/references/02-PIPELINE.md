# 执行框架（第二层）

> **作用**：通用的 4 步执行流程
> **加载时机**：Step 2（Pipeline 模式确定后）
> **注意**：Pipeline 是「不管做什么都一样的通用流程」

---

## Pipeline = 通用烹饪方法

```
不管你做的是鱼香肉丝还是番茄炒蛋，
烹饪的基本步骤是一样的：
1. 备料
2. 炒
3. 调味
4. 装盘

Pipeline 也是一样：
1. 计划
2. 执行
3. 汇总
4. 检查
```

---

## 4 步执行流程

### Step 1 — 计划

**问题**：我要做什么？

```
Pipeline Step 1：
1. 理解任务目标
2. 确定具体步骤
3. 预估时间和资源
```

### Step 2 — 执行

**问题**：按计划行动

```
Pipeline Step 2：
1. 按计划执行
2. 记录中间结果
3. 遇到问题及时调整
```

### Step 3 — 汇总

**问题**：我做出了什么？

```
Pipeline Step 3：
1. 整理所有产出
2. 汇总成统一格式
3. 准备最终输出
```

### Step 4 — 检查

**问题**：质量合格吗？

```
Pipeline Step 4：
1. 加载 QUALITY.md
2. 对照检查
3. 不合格则返回 Step 2 修正
```

---

## 门控机制

```
Step 1 ──→ Step 2 ──→ Step 3 ──→ Step 4
  │           │           │           │
  │    ⚠️ 前一步完成    │           │
  │        才能进入      │           │
  │           │           │           │
  └───────────┴───────────┴───────────┘
                     │
              只有全部通过
              才算完成任务
```

---

## 与 WORKFLOWS 的关系

**Pipeline 是「烹饪方法」，WORKFLOWS 是「菜谱」**

```
Pipeline（方法）        WORKFLOWS（内容）
─────────────────────────────────────────
热锅                  鱼香肉丝的配料
放油                  切肉丝
翻炒                  炒肉丝
调味                  加调料
出锅                  装盘
```

**Pipeline 不管你做什么菜，步骤都一样。**
**Pipeline 只负责「步骤」，不管「内容」。**
