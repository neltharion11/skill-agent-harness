# Context Workflow（上下文压缩工作流）

> **作用**：Pipeline 的「具体工作内容」层
> **适用场景**：长任务、上下文管理、记忆管理
> **加载时机**：Pipeline Step 2 执行时
> **组合**：Pipeline + context

---

## 关系说明

```
Pipeline（方法）         context（内容）
──────────────────────────────────────
Step 1: 计划      ←→  评估上下文
Step 2: 执行      ←→  压缩策略
Step 3: 汇总      ←→  建立快照
Step 4: 检查      ←→  验证连贯性
```

---

## 4 步压缩流程

### Step 1 — 上下文评估（对应 Pipeline 计划）

**问题**：当前上下文状态？

**评估指标**：
- 消息数量
- 上下文使用率
- 距溢出距离

**评估模板**：
```markdown
## 上下文评估

### 状态
- 消息数：{N}
- 使用率：{X%}
- 距溢出：{Y} 轮

### 可压缩部分
- 冗余讨论：{N1} 条
- 重复输出：{N2} 条
```

### Step 2 — 压缩策略（对应 Pipeline 执行）

**决策矩阵**：
| 使用率 | 策略 |
|--------|------|
| < 50% | 不压缩 |
| 50-70% | 渐进压缩 |
| 70-90% | 早期压缩 |
| > 90% | 强制压缩 |

**压缩操作**：
```markdown
### 保留
- 任务目标
- 用户偏好
- 关键决策

### 丢弃
- 冗余讨论
- 重复输出
- 已解决推理
```

### Step 3 — 建立快照（对应 Pipeline 汇总）

**问题**：如何记录压缩结果？

**快照模板**：
```markdown
## 任务快照 @ {时间}

### 任务目标
{目标}

### 已完成
- {项1}
- {项2}

### 进行中
- {当前项}

### 关键上下文
- {必须记住的事实}
```

### Step 4 — 验证（对应 Pipeline 检查）

**问题**：压缩后任务还连贯吗？

**检查清单**：
- [ ] 任务目标清晰
- [ ] 历史可追溯
- [ ] 当前步骤明确
