# 质量检查清单（第五层）

> **作用**：最终质量把关
> **加载时机**：Pipeline 最后一步（Step 4）

---

## 检查顺序

```
Step 1 ──→ Step 2 ──→ Step 3 ──→ Step 4
                              │
                              ▼
                         加载 QUALITY
                              │
                              ▼
                    检查输出是否合格
                              │
                    ┌─────────┴─────────┐
                    │                   │
                  合格               不合格
                    │                   │
                    ▼                   ▼
                任务完成           返回修正
```

---

## 通用检查项

### 完整性
- [ ] 所有必填章节都已完成
- [ ] 无空占位符
- [ ] 步骤之间有逻辑连贯性

### 格式一致性
- [ ] 符合模板要求
- [ ] 术语使用一致
- [ ] 结构清晰

### 内容质量
- [ ] 有实质内容（非占位填充）
- [ ] 逻辑连贯
- [ ] 结论有依据

---

## 按组合检查

### Pipeline + research
- [ ] 执行摘要完整（3-5 句）
- [ ] 每个发现有来源
- [ ] 置信度已标注
- [ ] 知识缺口已识别

### Pipeline + subagent
- [ ] 任务分解合理
- [ ] 子任务独立
- [ ] 结果汇聚完整

### Pipeline + context
- [ ] 快照记录完整
- [ ] 关键信息保留
- [ ] 任务连贯性验证通过

### Pipeline + analysis
- [ ] 评分矩阵完整
- [ ] 结论有数据支撑
- [ ] 建议可操作

```markdown
## 质量检查报告

```
